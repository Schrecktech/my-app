# Pre-Release Rename Plan

Checklist for renaming the app's package identifier and/or app name **before** the first App Store / Play Store release. Once published, changing these values creates a new app identity — users would need to reinstall. Do this before v1.0.0.

## 1. Rename the Package Identifier

Changes `com.scottschreckengaust.myapp` to a new identifier (e.g., `com.schrecktech.myapp`).

### Functional Files

| File | Field | Example |
|------|-------|---------|
| `app.json` | `expo.ios.bundleIdentifier` | `com.schrecktech.myapp` |
| `app.json` | `expo.android.package` | `com.schrecktech.myapp` |

### EAS Credentials

After changing the bundle ID / package name, existing EAS credentials (signing certs, provisioning profiles) are tied to the **old** identifier. You must re-configure them:

```bash
# iOS: set up new provisioning profile + cert for the new bundle ID
npx eas-cli credentials --platform ios

# Android: generate new keystore for the new package name
npx eas-cli credentials --platform android
```

### Documentation Updates

| File | What to update |
|------|---------------|
| `docs/DEVELOPER_GUIDE.md` | Simulator launch command (`xcrun simctl launch booted <new-id>`) |

## 2. Rename the App Name (`my-app` / `myapp`)

Changes the app name, slug, scheme, and all references. More impactful than the package identifier.

### Functional Files (build-breaking if missed)

| File | Field | Notes |
|------|-------|-------|
| `app.json` | `expo.name` | Display name shown under the app icon |
| `app.json` | `expo.slug` | EAS project identity (see warning below) |
| `app.json` | `expo.scheme` | Deep link scheme (`myapp://`) |
| `app.json` | `expo.ios.bundleIdentifier` | Update the `myapp` suffix |
| `app.json` | `expo.android.package` | Update the `myapp` suffix |
| `package.json` | `name` | npm package name |
| `constants/locales/en.ts` | `appName` | User-facing display name |

After editing `package.json`, regenerate the lockfile:

```bash
rm package-lock.json
npm install
```

### EAS Slug Warning

The `slug` field in `app.json` is the **EAS project identity**. Changing it disconnects you from the current EAS project (build history, OTA update channels, credentials). You have two options:

1. **Rename the EAS project** on [expo.dev](https://expo.dev) to match the new slug (preserves history)
2. **Create a new EAS project** with the new slug and update the `projectId` in `app.json` → `extra.eas.projectId`

### Owner Field

`app.json` → `expo.owner` is currently `"scottschreckengaust"`. If the Expo account owner is changing too (e.g., to a `schrecktech` org account), update this field and ensure you're logged into the correct account:

```bash
npx eas-cli login
npx eas-cli whoami
```

### Documentation Updates

These won't break the build but should be updated for accuracy:

| File | What to update |
|------|---------------|
| `.claude/CLAUDE.md` | Build output filename (`myapp-...ipa`) |
| `docs/DEVELOPER_GUIDE.md` | Build output filenames, simulator commands, clone URL |
| `docs/DEVELOPER_GUIDE.md` | Worktree examples (`../my-app-feature-name`) |
| `AGENTS.md` | Worktree examples |
| `docs/adr/0006-git-worktree-workflow.md` | Worktree path examples |

### Files That Do NOT Need Manual Changes

| File | Why |
|------|-----|
| `package-lock.json` | Regenerated by `npm install` |
| `eas.json` | References profiles, not names |
| `tsconfig.json` | No app name references |
| Source files (`app/`, `components/`, `hooks/`) | No hardcoded identifiers |
| `CHANGELOG.md` | Generated from git history |

## 3. Verification Checklist

After making all changes, verify everything still works:

- [ ] `npx tsc --noEmit` — type check passes
- [ ] `npx expo lint` — lint passes
- [ ] `npx expo start` — dev server starts, app loads in simulator/browser
- [ ] `npx eas-cli build --platform ios --profile preview --local` — iOS build succeeds
- [ ] `npx eas-cli build --platform android --profile preview --local` — Android build succeeds
- [ ] Deep links work with the new scheme (e.g., `newname://`)
- [ ] `gitleaks detect --source . --verbose` — no secrets leaked
- [ ] Commit with: `feat(build)!: rename app from my-app to <new-name>`

The `!` marks this as a breaking change since it changes the app identity.

---

## 4. Recommended Order

1. Decide on the new names (package ID, app name, slug, scheme)
2. Update `app.json` (all fields in one pass)
3. Update `package.json` name
4. Run `rm package-lock.json && npm install`
5. Update `constants/locales/en.ts` app name
6. Re-configure EAS credentials
7. Update documentation files
8. Run the verification checklist
9. Commit and tag
